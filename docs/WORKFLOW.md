# 개발 컨트롤타워 운영 규칙 (요약본)

## 원칙
- 구현(코딩)과 판단(우선순위/범위)은 분리한다.
- 컨트롤타워는 "다음 1개"만 결정한다.
- 정책(건강AI 코어 지침/화이트리스트)이 최우선이다.
- 추측 금지. 불명확하면 TBD로 남긴다.

## 매 작업 루틴(반복)
1) 목표 1줄: 이번 작업의 목적(무엇을 끝내면 성공인지)
2) 모드 확정: A(링크) / B(문서) / C(허브) 중 하나
3) 변경 범위: 수정/생성 파일 목록(최소화)
4) 안전장치: validator/renderer/정책 위반 여부 체크
5) 테스트: 추가/수정할 테스트 최소 1~3개 지정
6) 실행: Cursor로 반영 → npm test / npm run dev 확인
7) 기록: STATUS.md에 Done/Next 업데이트

## Project GPT 출력 포맷(고정)
- 목표(1줄)
- 작업 체크리스트(최대 7개)
- 수정/생성 파일 목록
- 테스트 목록
- 위험/주의 3개
- 완료 조건(검증 기준)
